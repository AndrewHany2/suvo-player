# Anti-Patterns Verdict

**Does this look AI-generated?** No. Suvo clears the product-register bar: one shared `Button`/`StatePanel`/`PosterCard` vocabulary, honest tonal-ladder depth instead of decorative glass, a restrained two-accent system, content-first composition. A user fluent in the best media apps would broadly trust it. What breaks the spell now is not fakeness but **self-inconsistency with its own design system** — the Single-Light Rule (indigo=active, cyan=focus, never at rest) is violated by Suvo's own shared components. **Verdict: Trustworthy, not slop — but it quietly breaks its own rulebook.**

# Overall Impression

The last pass fixed the loud tells (white/emoji Play CTA, off-ladder greys, silent Add-Channel, fake live progress) and the score rose 26 → 28. The critique now reaches the next layer: the app breaks its *own* Single-Light rule at rest in shared components (DiscoverPills, ContentShelf chevrons, LiveTV section icons all glow cyan unfocused), and the player — the most-used screen — is built from a bespoke button vocabulary instead of the shared kit. These are quieter but more corrosive: when cyan is everywhere at rest, cyan-on-focus stops meaning "you are here." Biggest opportunity: make cyan mean focus again, and make the player look built from the same parts as everything else.

# Priority Issues

**[P1] The design system violates its own Single-Light Rule at rest**
- **What:** DiscoverPills render indigo-tinted at rest (`accentAlpha(0.08)` bg, `accentAlpha(0.28)` border) with a **cyan** leading icon and **indigo** chevron, all unfocused (DiscoverPills.web:52-74). LiveTV section headers use a resting cyan `tv` icon (LiveTVScreen.web:286); ContentShelf title chevrons are resting cyan (ContentShelf.web:118). DESIGN.md: "Signal Cyan… appears only on focus or hover, never at rest."
- **Why it matters:** The exact "subtly-off" tell that makes a fluent user distrust the interface, and it dilutes the focus signal.
- **Fix:** Move resting glyphs/borders onto the steel/slate ladder (`muted`/`border`); reserve cyan for `:hover`/`isFocused`, indigo for active/selected. Pills at rest → `surface2` bg + hairline border, not indigo wash.
- **Suggested command:** `$impeccable colorize`

**[P1] Two competing button vocabularies**
- **What:** The player builds every control from ad-hoc inline styles (`S.iconBtn`, `S.playBtn`, `S.optionChip`, `S.closeBtn`) instead of the shared `Button` primitive that governs every other surface (VideoPlayerScreen.web:69-259). Product ban: "if the 'save' button looks different in two places, one is wrong."
- **Why it matters:** The most-used screen's buttons don't share the kit's focus-ring/glow language, so keyboard/remote focus feedback is inconsistent exactly where it matters most.
- **Fix:** Reconcile player controls to `Button` variants (or extract a shared `IconButton`) so focus/hover/active match the rest of the app.
- **Suggested command:** `$impeccable harden`

**[P2] Muted body text under 4.5:1 on elevated surfaces + faint used as an interactive glyph**
- **What:** `colors.muted` (#7A86A8) is ~5.3:1 on midnight bg but ~4.3:1 on `surface2` (#1B2236); the MovieDetail synopsis renders muted on a surface2 card (MovieDetail.web:343). `colors.faint` (#4A5575 ≈ 2.6:1) is misused as the **resting interactive** heart glyph in LiveCard (LiveTVScreen.web:144) and player footer hints — faint is spec'd placeholder/disabled only.
- **Why it matters:** Fails the stated WCAG 2.1 AA hard bar for real reading content and an interactive control.
- **Fix:** Use `text` (or a lightened steel) for body copy on surface2; the resting favorite heart should be `muted` at minimum.
- **Suggested command:** `$impeccable harden`

**[P2] Dishonest data undermines the honesty brand**
- **What:** CWCard shows a **fake 15% progress bar** when duration is unknown (`item.duration > 0 ? … : 15`, HistoryScreen.web:196); PosterCard/MyListCard show an **always-on "HD" badge** regardless of real quality (PosterCard.web:141, HistoryScreen.web:133). LiveCard *deliberately refuses* fake progress (LiveTVScreen.web:152-154) — the codebase already knows better and is inconsistent with itself.
- **Why it matters:** For a "calm, honest, your library is yours" voice, fabricated metadata is an avoidable trust leak Riley catches immediately.
- **Fix:** Show no bar when duration is unknown (match LiveCard); gate "HD" on real quality metadata or drop it.
- **Suggested command:** `$impeccable polish`

**[P2] Success feedback breaks the cinematic frame**
- **What:** Adding a custom channel triggers `globalThis.alert()` (LiveTVScreen.web:10-16,502) — a raw OS dialog inside a dark-theater UI. The error path is a proper inline banner; only success regressed to a native alert. (`zIndex.toast` exists in tokens.js:70 but nothing uses it.)
- **Why it matters:** Jarring break at a positive moment; inconsistent with the app's own inline handling.
- **Fix:** Build the missing toast tier (or a brief inline confirmation on the sheet) instead of `alert()`.
- **Suggested command:** `$impeccable polish`

**[P3] Icon-only top-bar actions with no labels**
- **What:** Accounts/Settings/Profile are icon-only (AppNavigator.web:310-359); Accounts uses a `signal` icon whose meaning isn't obvious.
- **Fix:** Add labels or accessible tooltips; pick a more literal Accounts glyph.
- **Suggested command:** `$impeccable clarify`
