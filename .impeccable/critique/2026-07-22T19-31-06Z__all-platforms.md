## Anti-Patterns Verdict

**Not AI slop — on any surface.** All three reviews independently reached the same verdict: a category-fluent user (Netflix/Plex/Jellyfin) would trust this. One tokenized Aurora language spans six targets with zero per-OS chrome, focus states are real, every loading/error/empty state routes through one StatePanel, and the code carries genuine cross-platform engineering (recovery machine, @supports aspect-ratio fallbacks, GPU-promoted spinners, portrait-lock lifecycle, deferred-undo).

**Deterministic scan:** 37 findings on the TV CSS — 1 warning, 36 advisory. Zero true slop antipatterns: no gradient text, no side-stripe borders, no ghost-cards, no over-rounding (max radius 14px; drift is toward *smaller* 4–6px, not larger), no decorative gradients/grids. The single warning (overused-font: Inter/Space Grotesk) is a **false positive** — both are deliberate documented DESIGN.md choices. The one substantive real pattern: **font-size drift off the documented ramp** (15px recurs ~8×, 17px ~4× across tvl.css + screen CSS) forming de-facto undocumented steps.

**No browser overlay** — auth-gated surface, not served; source + detector only.

## Overall Impression

This is the opposite of AI slop: a mature, disciplined, single-vocabulary system. The biggest opportunity is not concept — it's finishing the **scale-and-consistency layer** so the craft holds up off the 1920 reference and across the four modal patterns. Three cross-cutting themes dominate: (1) the uniform ss() scale model breaks legibility on non-reference viewports (web) and touch minimums (native); (2) component-vocabulary drift (modals on web, no-account copy on native, letter-bars on TV) undercuts the "one system" promise; (3) 10-foot legibility and the player control surface need tuning on TV.

## Priority Issues (cross-platform, ranked)

- **[P1] Web/Electron: no minimum type floor; UI scales uniformly with window width.** `ss()=round(n·clamp(width/1920,0.5,1.5))` clamps the *factor*, not resulting px — a 1440 laptop renders body text ~10px, a ~960px half-screen ~7px, with no reflow. The core desktop/Electron persona rarely sits at 1920. *Fix:* clamp type (and touch targets) to legible minimums or add 2–3 reflow breakpoints; decouple font-size from the uniform factor. `src/utils/scaleSize.js:70`. → **$impeccable adapt**
- **[P1] Native: `sm` icon-only buttons fall below 44×44.** `sm` has `minH:0` and only vertical hitSlop; width = `ss(44)` → ~37px at the 0.85 small-phone clamp, with no horizontal slop. Hits the player row, channel-zap arrows, Accounts actions. *Fix:* symmetric hitSlop + unscaled `minWidth:44` on icon-only sm. `src/ui/Button.native.jsx:25-29,85`. → **$impeccable adapt**
- **[P1] Web: four modal patterns, no shared primitive.** native `<dialog>` (Accounts/Settings), hand-rolled fixed div (exit, r16), RN bottom-sheet `<Modal>` (Add-Channel), absolute YStack (confirm, r20) — backdrops/radii/entrance all drift; explicit "inconsistent component vocabulary" ban. *Fix:* one Aurora Modal primitive; migrate all call sites. `src/navigation/AppNavigator.web.jsx:833`. → **$impeccable harden**
- **[P1] Native: no-account empty state drifts AND ships "IPTV" to users.** Movies/Series/LiveTV hand-roll copy while History uses LABELS; `labels.js:37` noAccountBody contains "IPTV" against the documented store-compliance constraint. *Fix:* route all four through LABELS.noAccount*; drop "IPTV". `src/ui/labels.js:36-39`. → **$impeccable clarify**
- **[P1] TV: browse/Accounts text ships at arm's-length sizes, under the screen's own 10-foot ramp.** 15px poster/channel titles + 13–14px Accounts copy contradict the file's own `--a-fs-card=22 / --a-fs-body=18` tokens — reads as half-converted from web. *Fix:* raise to the declared ramp and route through `--a-fs-*`. `src/styles/tvl.css` + `*.tv.css`. → **$impeccable typeset**
- **[P1] TV: player control band overflows into a dense wrapping chip wall.** 9 labeled 56px chips at ~150px each exceed 1280 and wrap over the video — the cluttered-UI anti-reference; buries subtitles/audio among brightness/stats. *Fix:* keep Subtitles/Audio/Speed/Fit inline, collapse the rest behind "More" (matching web/native). `src/screens/VideoPlayerScreen.tv.jsx:517-621`. → **$impeccable distill**
- **[P2] Web+Native: typography/type-face inconsistencies.** Detail hero titles render in Inter not Space Grotesk (web); display tracking under the -0.04em floor. *Fix:* `fontFamily=fonts.display` on detail heroes + Movies CategoryPage heading. `src/components/MovieDetail.web.jsx:183`. → **$impeccable typeset**
- **[P2] Web: series trailer autoplays with sound.** MovieDetail appends `mute=1`; SeriesDetail omits it — bursts audio in a quiet room, contradicting the app's own intent. *Fix:* add `&mute=1`. `src/components/SeriesDetail.web.jsx:538`. → **$impeccable polish**
- **[P2] Native: player Close is top-left only; forms lack return-key submit.** Top-left exit is a hard landscape reach with no iOS back-swipe; forms need returnKeyType/onSubmitEditing/focus advance. → **$impeccable adapt**
- **[P2] TV: password reveal unreachable by remote; top-row focus ring clips; hidden letter-jump.** Eye toggle is onClick-only (not in D-pad index); first-row 4px ring+offset clips at scrollTop 0 (add scroll-padding-top); FF/REW ±5 jump has no on-screen cue. → **$impeccable harden**
- **[P2] Web: player volume is a bare native `<input type=range>`** next to the custom Aurora seek bar — breaks the control vocabulary. *Fix:* Aurora-styled volume slider. `src/screens/VideoPlayerScreen.web.jsx:1134`. → **$impeccable polish**

## Persona Red Flags

- **Cross-device viewer (phone→desktop→TV):** the promise ("Synced across your devices", Continue Watching) lands on all three, but each surface has a different weak link — small text on <1920 desktop, small text at 10ft on TV, and modal/type-face drift makes "same product" wobble between surfaces.
- **One-handed / on-the-go (native):** top-left player Close (worst in landscape), ~37–41px icon controls, and no return-key form flow.
- **Privacy-conscious / quiet-room:** series trailer bursts audio (web); "IPTV" still in user copy (native); DeviceLocked/ConfigError dead-end at "contact support" with no path.
- **Small-window desktop:** 6–10px text with no reflow; half-screen multitasking halves all text.

## Minor Observations

- **Verified cross-cutting:** both `Button.web` and `Button.native` use `radii.sm` (8px), but DESIGN.md specifies buttons at `rounded.md` (14px) — either update the doc or the components.
- Always-on indigo LIVE dot on every resting channel card (web + native) — borderline against the Single-Light "no resting tint" rule.
- Raw emoji avatar in the native Aurora header (`AppNavigator.jsx:87`) — small slop tell against tokenized chrome.
- TV back chevron is indigo-tinted at rest; `.tvl-topbar-back` 24px is off even the TV ramp.
- TV radius drift 4/5/6px and `rgba(255,255,255,.)` white overlays — genuine but low-impact drift.
- ProxiedImage placeholder uses `colors.faint` (spec'd placeholder/disabled only) — very dim against surface.
- Native pills pass `animation="quick"`/`hoverStyle` that primitives.native explicitly drops — harmless dead props.
