## AI Slop Verdict
Not slop — this is a bespoke, token-disciplined 10-foot surface with genuine webOS-Chromium-71 awareness (aspect-ratio @supports fallbacks, compositor-promoted spinner, literal-hex CSS with no var() in the static sheet, focus fill instead of a stripped glow halo). A Linear/Stripe-fluent user would trust the shelves, posters, detail banners, and the consistent 4px cyan ring. Two things make that user pause, and they are behavioral not cosmetic: (1) pressing OK on a prominent CTA/Retry button does nothing on several screens because those StatePanel buttons are never wired into the D-pad routers, and (2) the History movie-detail navigates a visibly-horizontal button row with Up/Down while the identical row on Movies uses Left/Right. The visual system reads as earned-familiar; the interaction wiring has holes that a remote-only viewer hits immediately.

## Cognitive Load
Intrinsic load is well-managed on the shelf/poster surfaces (content-forward, chrome recedes). Extraneous load spikes in three places: the Movies/Series category flow stacks four Up/Down zones (topbar-back → search → letter-filter → grid) with no persistent 'you are here' indicator beyond the ring; the alpha filter presents 27 options in a horizontal scroller (well over the >4 flag) with single-step traversal; and the account row's connect/Edit/Delete tri-state column model isn't visually distinguished from the row-level highlight. Missing-state gaps: grid initial-load is a spinner not a skeleton (shelves do it right), and the interactive states of StatePanel (its CTA/Retry) have no focus/active representation on TV at all.

## Emotional Journey
Peak moments are strong: the cinematic Home hero ('Synced across your devices'), the resume progress bars, the ContinueCard focus play-glyph, and the busy overlay that keeps the last decoded frame visible while reconnecting ('Reconnecting…') — that last one is exactly the reassurance a flaky-network viewer needs at a high-stakes moment. The valleys are all the same shape: the viewer is asked to act and cannot. First-run with no account shows 'Connect account' → OK does nothing. A category fetch fails → 'Retry' → OK does nothing. Worst of all, the stream dies mid-watch, StatePanel says 'Failed to load stream / Retry', and the only working key is Back. The end-state of an error session is therefore a shrug, which poisons the peak-end memory precisely where trust is most fragile.

## Strengths
- Genuine old-Chromium engineering, not theater: @supports not(aspect-ratio) ratio-box fallbacks for poster/logo boxes, translateZ compositor promotion on the spinner so a catalog JSON.parse can't freeze it, literal-hex tokens with no var() in the patch-inlined sheet, and a documented no-canvas-snapshot busy overlay that exploits the hardware video plane retaining the last frame.
- Disciplined Single-Light adherence in the details: resume/progress fills are indigo (accent) while cyan is reserved for focus, the LIVE badge is success-green (not the active-path indigo), episode badges sit on the surface2 ladder, all with inline comments explaining the token choice.
- One canonical focus treatment (4px cyan outline + offset + 0.20 cyan fill, thickened to 5px on webOS/tizen) applied across cards, buttons, seasons, episodes and letter chips — legible at distance and consistent, plus VirtualShelvesTV has a dev-guard invariant ensuring the focused card is always mounted.

## Priority Issues

### [P0] StatePanel CTA buttons (Connect account / Browse Movies) are unreachable by the remote
- **File:** src/ui/StatePanel.jsx (consumed unrouted by src/screens/LiveTVScreen.tv.jsx:358, MoviesScreen.tv.jsx:415, HistoryScreen.tv.jsx:493)
- **Why:** On every no-account/empty screen the primary CTA is a StatePanel <Button>, but each screen's keydown/register router only handles zone/grid nav and Up→navbar/Back — none routes focus or Enter to the StatePanel button. Enter lands on document.body and is preventDefault'd, so OK does nothing. The first-time-setup persona lands on Movies/Live/History with no account, sees 'Connect account', presses OK, and nothing happens; they can only stumble to the global navbar.
- **Fix:** Make StatePanel's CTA/Retry a first-class D-pad target: either give each screen's router an explicit 'panel' zone that focuses and activates the StatePanel button on Enter, or render the empty/error CTA as a screen-owned focusable button (tvl-btn with tvl-btn--on) inside the existing focus model. Verify on sim:lg that OK fires it.
- **Command:** $impeccable harden

### [P0] Retry is dead on both load errors and fatal playback errors
- **File:** src/screens/VideoPlayerScreen.tv.jsx:712 (also LiveTVScreen.tv.jsx:411, MoviesScreen.tv.jsx:402, SeriesScreen.tv.jsx:746)
- **Why:** LiveTV/Movies/Series render StatePanel mode='error' with onRetry, but handleChKey/routeDir short-circuit (e.g. LiveTVScreen.tv.jsx:316 `if (!pg?.items) return`, and the top-level error branches sit above VirtualShelvesTV so no router is mounted) so OK never reaches Retry. Most damaging: VideoPlayerScreen fatal overlay (isFatal) shows 'Failed to load stream / Retry' + a Close Button, but the keydown handler only maps Back→handleClose — Retry has no key path. The flaky-network viewer whose stream drops mid-watch cannot retry, only exit.
- **Fix:** Route Enter to onRetry in each error state; in VideoPlayerScreen add explicit D-pad focus between the Retry and Close buttons in the fatal overlay (they are the only interactive elements in that state) with a visible ring, defaulting focus to Retry.
- **Command:** $impeccable harden

### [P1] History movie-detail navigates a horizontal button row with Up/Down
- **File:** src/screens/HistoryScreen.tv.jsx:304
- **Why:** handleMovieDetailKey maps KEY_UP/KEY_DOWN to btnIdx-1/+1, but the buttons render in .tvl-det-hero-btns which is display:flex (a horizontal row, identical markup to MoviesScreen). MoviesScreen's handleDetailDir correctly uses Left/Right for that same row. So the identical-looking Play/From-Start/Trailer/My-List row responds to opposite axes depending on which screen opened it — the viewer presses Right expecting to move to Trailer and nothing happens.
- **Fix:** Make HistoryScreen's movie-detail use Left/Right between the action buttons and Up/Down only for row↔topbar-back↔navbar, matching MoviesScreen.tv.jsx handleDetailDir exactly.
- **Command:** $impeccable clarify

### [P1] Source-type toggle draws the cyan focus ring on the SELECTED option at rest
- **File:** src/screens/AccountsScreen.tv.css:136
- **Why:** In the Add/Edit form the selected type is `tvl-btn tvl-btn--on`, and .tvl-btn--on is `outline: 4px solid var(--a-cyan)` (AccountsScreen.tv.css:136). Cyan is the focus-only light per the Single-Light rule, so the selected segment permanently looks focused even when the toggle isn't; meanwhile the actual toggle-focus cue is only the near-invisible rgba(255,255,255,0.03) field wash. The same class means 'focused' on the Save/Cancel buttons, so its meaning is overloaded (selection vs focus).
- **Fix:** Give the selected segment an indigo (active-path) treatment — indigo fill/border, accentText — and reserve the cyan .tvl-btn--on strictly for the focused element; introduce a distinct focus indicator for the toggle field that matches the canonical ring.
- **Command:** $impeccable colorize

### [P1] Delete button label is red-on-navy below AA contrast
- **File:** src/screens/AccountsScreen.tv.css:74
- **Why:** .tvl-acc-btn-danger keeps the .tvl-acc-btn background (var(--a-border) #28324E) and sets color to var(--a-danger) #E5484D. Red text on dark navy is roughly 3.1:1 — under the 4.5:1 bar for 14px body text. At 10 feet the destructive action is the hardest label to read, which is both an a11y failure and a safety concern.
- **Fix:** Either raise the danger label to a lighter red that clears 4.5:1 on #28324E, or give the Delete button a filled danger background with #EAF0FF/near-white text at rest (keeping the confirm dialog as the actual safety gate).
- **Command:** $impeccable colorize

### [P2] Account row shows two focus rings simultaneously (row + Edit/Delete)
- **File:** src/screens/AccountsScreen.tv.jsx:447
- **Why:** An account row gets .tvl-acc-item--on whenever focus===rowIdx regardless of col, and Edit/Delete get .tvl-acc-btn--on when col is 1/2. So moving Right to Edit paints the canonical ring on the button while the whole row still carries its own ring+fill — two competing cyan rings at once, and the col===0 'connect the whole row' state is not visually distinct from the row-level highlight.
- **Fix:** When col>0, drop the row-level --on so only the focused button rings; give col===0 an explicit 'row focused, will connect' affordance distinct from the button rings.
- **Command:** $impeccable clarify

### [P2] Form field focus indicator is inconsistent and faint vs the canonical ring
- **File:** src/screens/AccountsScreen.tv.css:133
- **Why:** .tvl-field--on is `background: rgba(255,255,255,0.03)` plus a 2px cyan input border, while every other focusable element on TV uses the 4px cyan outline + 0.20 fill (tvl.css:565). At 10 feet the field wash is nearly invisible and the 2px border is thinner than the established ring, so 'which field am I on' is the weakest focus cue in the app right where accurate input matters.
- **Fix:** Apply the canonical focus ring (4px outline + offset + focus-fill) to the focused field wrapper so text-entry focus matches the rest of the surface.
- **Command:** $impeccable clarify

### [P2] A LIVE badge on every channel tile creates a badge-wall
- **File:** src/screens/LiveTVScreen.tv.jsx:523
- **Why:** ChannelCard always renders `<span className='tvl-ch-live'>LIVE</span>` (LiveTVScreen.tv.jsx:523). In a 6-column grid every tile carries an identical bright-green badge, which is pure redundancy (all channels are live) and pushes toward the 'cluttered logo-wall EPG' anti-reference the brief explicitly bans.
- **Fix:** Drop the per-tile LIVE badge (or reserve a live indicator only for the currently-playing channel); let the logo + name carry the tile.
- **Command:** $impeccable distill

### [P2] Alpha filter: 27 options, single-step traversal, no fast jump
- **File:** src/screens/MoviesScreen.tv.jsx:544
- **Why:** ALPHA is ALL + A–Z (27 chips) navigated one at a time with Left/Right in the 'filter' zone. Reaching Z from ALL is 26 presses, and it's a >4-option decision point rendered as 36px chips. For a large-catalog browse this is the least efficient path in the app.
- **Fix:** Support page/skip stepping (e.g. FF/REW or holding Right to accelerate), or collapse to a smaller set of ranged buckets; enlarge the active chip so the current letter reads from the couch.
- **Command:** $impeccable optimize

### [P2] Grid initial-load uses a spinner while shelves use skeletons
- **File:** src/screens/MoviesScreen.tv.jsx:558
- **Why:** Movies/Series/LiveTV grid loads render StatePanel mode='loading' (a spinner), but VirtualShelvesTV renders SkeletonPoster rows. The brief calls for skeletons over spinners, and the mixed treatment means the same 'content is coming' status looks different depending on layout mode.
- **Fix:** Render a skeleton poster grid for the PagedGridTV loading state to match the shelf skeletons and the design guidance.
- **Command:** $impeccable animate

### [P3] Detail title uses font-weight:900 outside the token ramp
- **File:** src/styles/tvl.css:462
- **Why:** .tvl-det-hero-title is font-weight:900 but fontWeights tops out at 700 (bold) and Space Grotesk may not ship a 900 face on the TV fallback, yielding a synthesized/again-700 weight that won't match the intended emphasis and drifts from the type system.
- **Fix:** Use the token bold (700) or add a defined heavy weight to the ramp; keep display weights within the documented set.
- **Command:** $impeccable distill

### [P3] Seasons and episodes are non-semantic <div>s with no role
- **File:** src/screens/SeriesScreen.tv.jsx:884
- **Why:** .tvl-season-btn and .tvl-episode are plain <div> elements driven entirely by the custom D-pad handler, with no role='button' or aria-label. They work for sighted remote users but expose nothing to assistive tech, unlike the poster cards which do set role/aria-label.
- **Fix:** Add role='button' and an aria-label (season number / episode title) to these interactive divs to match the poster/card pattern.
- **Command:** $impeccable harden

## Persona Red Flags
- Remote-only TV viewer: the recurring dead-button pattern (StatePanel CTA/Retry with no D-pad route) means the viewer is repeatedly asked to act and physically cannot — a mouse-era affordance stranded on a remote-only surface.
- First-time playlist setup: lands on a content screen with no account, sees 'Connect account', presses OK, nothing happens — the single most important first-run action is unreachable without discovering the global navbar.
- Viewer on a flaky network: the worst-case is fully realized — when the stream dies mid-watch the fatal overlay's 'Retry' is not wired to any key, so the only escape is Back, turning a recoverable blip into a session-ending failure at the highest-stakes moment.

## Minor Observations
- Hero.tv.jsx appears to be a dormant path (Home passes renderHero, Movies/Series pass showHero={false}) — dead-ish code that risks drift from the interactive renderHomeHero it mirrors.
- VideoPlayer close/next/start-over buttons use accentAlpha(0.9) indigo as a resting background; defensible as on-video 'active path' chrome, but it's a heavy full-saturation accent at rest that brushes the Single-Light guidance.
- 'Need help connecting? Visit suvo.app/help' is plain 14px muted text and non-actionable on a TV (no way to open a URL) — informational only.
- Meta/hint text (13px hints, .tvl-det-tag 15px, seekHint 16px) sits near the pre-scale legibility floor; fine if the 1280→×1.5 upscale holds, but there's no on-device slack if it doesn't.
- ContinueCard.tv centers its focus play-glyph with transform: translate(-50%,-50%); harmless on Chromium 71 but a flex-centered absolute box would avoid a transform the design language otherwise minimizes.
- onLoadMore is passed to VirtualShelvesTV from Movies/Series but is a documented no-op stub — confirm it stays intentionally inert so a future reader doesn't wire it expecting paging.
